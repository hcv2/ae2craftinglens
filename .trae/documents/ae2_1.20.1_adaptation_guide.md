# AE2 1.20.1 Forge 完整适配指南

## 概述
本文档记录了将 AE2 Crafting Lens 适配到 AE2 1.20.1 Forge 版本的完整过程和关键技术点。

## 核心源码位置（AE2 forge/1.20.1 分支）

### CPU 核心实现
- **CraftingCPUCluster**: `src/main/java/appeng/me/cluster/implementations/CraftingCPUCluster.java`
  - 包含 `logic` 字段（CraftingLogic 类型）
  
- **CraftingLogic**: `src/main/java/appeng/me/cluster/implementations/CraftingLogic.java`
  - 包含 `job` 字段（CraftingJob 类型）
  
- **CraftingJob**: `src/main/java/appeng/me/cluster/implementations/CraftingJob.java`
  - 包含 `tasks` 映射（任务数据）

### 网络服务接口
- **ICraftingService**: `src/main/java/appeng/api/networking/crafting/ICraftingService.java`

## Access Transformer 配置

### 1.20.1 Forge 特有的 AT 配置
在 `src/main/resources/META-INF/accesstransformer.cfg` 中精确定义：

```cfg
# Access Transformer for AE2 Crafting Lens (AE2 1.20.1 Forge)
# Makes private fields accessible at runtime

# CraftingCPUCluster fields - Core CPU implementation
public appeng.me.cluster.implementations.CraftingCPUCluster logic
public appeng.me.cluster.implementations.CraftingCPUCluster f_legacy_logic_

# CraftingLogic fields - Job management (AE2 1.20.1 specific)
public appeng.me.cluster.implementations.CraftingLogic job

# CraftingJob fields - Task data (AE2 1.20.1 specific)
public appeng.me.cluster.implementations.CraftingJob tasks

# CraftingNode - Pattern access
public-f appeng.me.cluster.implementations.CraftingNode getPattern()Lappeng.api.crafting.IPatternDetails;

# CraftingStatusMenu fields - UI integration
public appeng.menu.me.crafting.CraftingStatusMenu selectedCpu

# CraftingCPUMenu fields - CPU menu
public appeng.menu.me.crafting.CraftingCPUMenu cpu

# PatternProviderLogic host field (for ExtendedAE and other addons)
public appeng.helpers.patternprovider.PatternProviderLogic host
```

**重要说明**：
- 1.20.1 使用传统的 Forge AT 格式
- 如果不配置 AT，反射会报 `NoSuchFieldException`
- 字段名在源码中是 `logic`，但在生产环境中需要 AT 才能访问

## 网络协议（Networking）

### 1.20.1 Forge 网络处理

**注册方式**：
- 使用 `SimpleChannel` 和 `NetworkRegistry.newSimpleChannel`
- 通道 ID：`ae2craftinglens:main`

**线程安全**：
```java
public static void handle(RequestPatternProvidersPacket packet, Supplier<NetworkEvent.Context> contextSupplier) {
    NetworkEvent.Context context = contextSupplier.get();
    // 必须使用 enqueueWork 确保在主线程运行
    context.enqueueWork(() -> PatternProviderRequestHandler.handle(packet, context));
    context.setPacketHandled(true);
}
```

**重要**：必须使用 `NetworkEvent.Context.enqueueWork` 来确保代码在主线程运行，否则访问 `ICraftingService` 时可能会触发 `ConcurrentModificationException`。

## 渲染阶段

### RenderLevelStageEvent（1.20.1）

在 Forge 1.20.1 中，渲染高亮使用 `RenderLevelStageEvent`：

```java
@SubscribeEvent
public static void onRenderLevelStage(RenderLevelStageEvent event) {
    if (event.getStage() == RenderLevelStageEvent.Stage.AFTER_PARTICLES) {
        // 渲染高亮框
        // 注意：需要从 PoseStack 中减去镜头的位移（Camera Position）
    }
}
```

**重要注意事项**：
- 选择 `AFTER_PARTICLES` 阶段
- 需要从 `PoseStack` 中减去镜头的位移（Camera Position）
- 否则线框会固定在屏幕坐标而不是世界坐标

## 适配总结清单

### ✅ 1. 检查 Mappings
确保在 `build.gradle` 中使用正确的映射：
```groovy
// 使用 parchment 或最新的 official 映射
mappings channel: 'official', version: minecraft_version
// 或
mappings channel: 'parchment', version: '2023.09.03-1.20.1'
```

这决定了你在开发环境下看到的字段名是否为 `logic`。

### ✅ 2. 配置 AT
这是解决 `NoSuchFieldException` 的**唯一优雅方案**。

必须在 `src/main/resources/META-INF/accesstransformer.cfg` 中声明所有需要访问的私有字段。

### ✅ 3. 双端解耦
- **客户端**（`CraftingScreenHandler`）：
  - 只负责发包（发送点击的物品 AEKey）
  - 不执行任何反射计算
  
- **服务端**（`PatternProviderRequestHandler`）：
  - 负责所有的反射计算和 Job 扫描
  - 计算完结果后回传给客户端
  
- **客户端渲染**（`PatternProviderHighlightRenderer`）：
  - 只负责接收结果并渲染高亮

### ✅ 4. AEKey 序列化（AE2 15.4.10+）

使用 AE2 15.4.10+ 的正确序列化方式：

**编码**：
```java
// AE2 15.4.10+: 使用 AEKey 的类型进行序列化
Class<?> aeKeyClass = Class.forName("appeng.api.stacks.AEKey");
Method getTypeMethod = aeKeyClass.getMethod("getType");
Object keyType = getTypeMethod.invoke(packet.what);

Class<?> keyTypeClass = Class.forName("appeng.api.stacks.AEKeyType");
Method streamMethod = keyTypeClass.getMethod("stream");
Object keyStream = streamMethod.invoke(keyType);

Method writeMethod = keyStream.getClass().getMethod("write", FriendlyByteBuf.class, aeKeyClass);
writeMethod.invoke(keyStream, buffer, packet.what);
```

**解码**：
```java
// AE2 15.4.10+: 使用 AEKeyType 的 stream 方法反序列化
Class<?> keyTypeClass = Class.forName("appeng.api.stacks.AEKeyType");
Method streamMethod = keyTypeClass.getMethod("stream");
Object keyStream = streamMethod.invoke(null);

Method readMethod = keyStream.getClass().getMethod("read", FriendlyByteBuf.class);
Object what = readMethod.invoke(keyStream, buffer);
```

## 常见问题排查

### 问题 1: NoSuchFieldException
**原因**：缺少 Access Transformer 配置

**解决**：
1. 检查 `accesstransformer.cfg` 是否正确配置
2. 确保字段名与源码一致
3. 重新构建模组

### 问题 2: ConcurrentModificationException
**原因**：在非主线程访问 ICraftingService

**解决**：
```java
context.enqueueWork(() -> {
    // 所有访问 AE2 API 的代码都在这里执行
});
```

### 问题 3: NoSuchMethodException (AEKey 序列化)
**原因**：使用了错误的 AEKey 序列化方法

**解决**：
- AE2 15.4.10+ 使用 `AEKey.getType().stream().write()` 和 `AEKeyType.stream().read()`
- 不要使用已弃用的 `writeToPacket()` 或 `fromPacket()`

### 问题 4: 高亮框位置不正确
**原因**：未减去 Camera Position

**解决**：
```java
// 从 PoseStack 中减去镜头位移
PoseStack poseStack = event.getPoseStack();
Camera camera = mc.gameRenderer.getMainCamera();
Vec3 cameraPos = camera.getPosition();
poseStack.translate(-cameraPos.x, -cameraPos.y, -cameraPos.z);
```

## 总结

将 AE2 Crafting Lens 适配到 AE2 1.20.1 Forge 版本需要注意以下关键点：

1. **Access Transformer** - 正确配置 AT 是访问 AE2 内部 API 的关键
2. **类结构变化** - 使用正确的 1.20.1 类名（CraftingLogic, CraftingJob）
3. **网络线程安全** - 使用 `enqueueWork` 确保主线程执行
4. **AEKey 序列化** - 使用 AE2 15.4.10+ 的新序列化机制
5. **渲染坐标** - 减去 Camera Position 确保正确的世界坐标

遵循本指南可以确保模组在 AE2 1.20.1 中稳定运行。
